
# Spring Cloud Microservices – Interview Questions (Quick Revision)

## 1. Spring Cloud Basics
- What is Spring Cloud and why is it used?
- Explain the difference between Spring Boot and Spring Cloud.
- What problems do microservices solve?

## 2. Config Server
- How does Spring Cloud Config Server work?
- What is the purpose of `spring.config.import`?
- What happens if the Config Server is down?

## 3. Eureka Naming Server
- What is service discovery?
- Difference between client-side and server-side discovery?
- What does Eureka’s self-preservation mode do?

## 4. API Gateway
- What is the purpose of API Gateway?
- Difference between Zuul and Spring Cloud Gateway?
- Explain filters in API Gateway.

## 5. Feign Client
- What is Feign and why is it used?
- How does Feign communicate with Eureka?
- Common Feign issues and how to debug them.

## 6. Resilience4j
- What is a circuit breaker?
- Explain Retry, RateLimiter, Bulkhead patterns.
- How do you configure fallback methods?

## 7. Distributed Tracing
- What is Sleuth?
- How does Zipkin help in tracing?
- How do trace IDs propagate?

## 8. Microservice Design
- How do you split a monolithic application into microservices?
- What is bounded context?
- How do you handle communication between microservices?

## 9. Docker & Deployment
- What is containerization?
- Difference between Docker image and container?
- How do you run multiple microservices using Docker Compose?

## 10. General Architecture Questions
- What is the difference between synchronous and asynchronous communication?
- How do you ensure observability in microservices?
- How do you handle versioning in microservices?
